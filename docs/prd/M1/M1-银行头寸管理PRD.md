## 三、业务规则

1. **实时刷新**: 头寸数据按配置的频率自动刷新
2. **限额检查**: 交易提交时检查账户限额，超限阻止提交
3. **预警触发**: 余额低于预警阈值时自动触发预警
4. **在途资金**: 计算可用余额时需扣除在途资金
5. **数据权限**: 用户只能看到有权限的业务单元头寸
6. **归集联动**: 归集账户余额变化自动触发调拨

---

## 四、头寸计算

### 4.1 可用余额计算

```
可用余额 = 账户余额 - 冻结金额 - 在途金额 - 预留限额
```

### 4.2 头寸汇总

```
集团头寸 = Σ(各业务单元头寸)
业务单元头寸 = Σ(所属账户余额)
币种头寸 = Σ(该币种账户余额)
银行头寸 = Σ(该银行账户余额)
```
