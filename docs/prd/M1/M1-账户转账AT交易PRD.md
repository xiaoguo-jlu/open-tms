## 一、模块概述

**模块名称**: transfer - 账户转账管理  
**功能定位**: 管理企业银行账户之间的资金划转，包括内部转账和跨行转账，支持审批和执行跟踪  
**用户角色**: 资金管理人员、出纳、资金经理

## 五、业务规则

1. **编号规则**: 转账编号 = TR + 日期 + 序号，如 `TR202604060001`
2. **金额校验**: 转账金额需大于0，精度匹配币种小数位数
3. **账户校验**: 付款账户和收款账户不能相同
4. **余额检查**: 付款账户余额需大于转账金额
5. **限额检查**: 转账金额不能超过账户日间/日终限额
6. **授权规则**: 可配置金额阈值，超过阈值需授权
7. **跨行转账**: 跨行转账需额外校验银行通道可用性
8. **跨币种转账**: 需检查汇率有效性
9. **数据权限**: 用户只能操作有权限的账户

## 六、验收标准

| 功能 | 验收条件 |
|------|----------|
| 新建转账 | 字段必填校验，提交成功 |
| 暂存 | 暂存后可在列表中找到，状态为New |
| 审批授权 | 审批通过后状态更新为Authorized |
| 执行转账 | 执行后状态正确更新 |
| 执行失败 | 失败记录原因，可重试 |
| 查询导出 | 多条件组合查询，Excel导出正常 |
| 支付模板 | 模板可创建/使用/管理 |
| 权限控制 | 无权限账户不可操作 |
