## 一、模块概述

**模块名称**: organization - 组织架构与权限管理  
**功能定位**: 提供组织架构管理、用户角色权限控制，支持多业务单元、多角色的资金管理系统基础权限体系  
**用户角色**: 系统管理员、IT运维人员、资金管理人员  
**国际化**: 支持中英文界面，所有界面文字需支持多语言切换

## 三、业务规则

1. **编码唯一性**: 业务单元编码、用户编号、登录账号、角色编码均需唯一
2. **级联关系**: 
   - 删除业务单元时需检查关联用户
   - 删除角色时需检查关联用户
3. **密码规则**: 密码长度至少8位，包含字母和数字
4. **数据权限**: 用户登录后根据角色自动过滤可访问数据
5. **权限校验**: 所有API需校验用户功能权限，无权限返回403
6. **超级管理员**: 系统内置超级管理员角色，拥有全部权限

## 四、验收标准

| 功能 | 验收条件 |
|------|----------|
| 业务单元管理 | 支持CRUD，层级关系正确 |
| 用户管理 | 登录认证，密码加密，重置密码正常 |
| 角色管理 | 权限配置生效，按钮级控制正确 |
| 功能权限 | 无权限菜单/按钮隐藏或不可用 |
| 数据权限 | 数据查询结果符合权限规则 |
| 用户角色关联 | 多角色权限取并集 |

## 五、技术说明

- 用户密码使用BCrypt加密存储

- 数据权限在查询层统一拦截注入
- 登录日志记录IP地址和时间
