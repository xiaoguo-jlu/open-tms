## 一、模块概述

**模块名称**: approval - 交易审批流程  
**功能定位**: 提供可配置的多级审批流程，支持条件分支、加签、转交等审批场景  
**用户角色**: 交易员、交易主管、资金经理、审批人

## 三、审批流程说明

### 3.1 流程触发

```
交易提交 → 匹配流程模板 → 按条件跳转节点 → 审批人审批 → 下一节点/结束
```

### 3.2 审批状态流转

```
待审批 → 审批中 → 已通过/已驳回
           ↓
        已撤回
```

### 3.3 会签/或签逻辑

- **会签**: 所有审批人全部通过，才进入下一节点
- **或签**: 任一审批人通过，即可进入下一节点

### 3.4 条件分支示例

```
金额 < 10万 → 节点1(资金经理审批)
10万 <= 金额 < 100万 → 节点2(部门主管审批)
金额 >= 100万 → 节点3(资金总监审批) → 节点4(CEO审批)
```

## 四、业务规则

1. **流程匹配**: 提交时按"适用业务+适用机构"匹配流程模板
2. **审批权限**: 审批人必须是流程中配置的用户或角色成员
3. **顺序审批**: 默认按节点顺序逐级审批
4. **驳回规则**: 驳回后交易回到草稿状态，可修改后重新提交
5. **超时处理**: 超时后可自动通过或升级到上级
6. **数据权限**: 审批人只能看到有数据权限的交易
