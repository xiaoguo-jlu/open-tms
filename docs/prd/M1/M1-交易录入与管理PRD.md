## 一、模块概述

**模块名称**: dealing - 交易录入与管理  
**功能定位**: 支持资金交易的手工录入、暂存、提交、查询、变更、撤销等全生命周期管理  
**用户角色**: 交易员、交易主管、资金经理

---

## 二、交易类型定义

| 类型 | 子类型 | 说明 | 优先级 |
|------|--------|------|--------|
| **存款** | 定期存款、活期存款、大额存单、通知存款 | 资金存放银行 | P0 |
| **贷款** | 短期贷款、中长期贷款、信用贷款、抵押贷款 | 从银行借入资金 | P0 |
| **外汇** | 即期外汇、远期外汇、NDF、外汇掉期 | 外汇交易 | P0 |
| **同业** | 同业存放、存放同业、同业拆借 | 同业资金往来 | P0 |

## 四、业务规则

1. **编号规则**: 交易编号 = 业务单元编码 + 日期 + 序号，如 `BJ202604050001`
2. **金额校验**: 交易金额需大于0，精度匹配币种小数位数
3. **日期校验**: 
   - 起息日需大于等于当前日期
   - 到期日需大于起息日
   - 交割日需跳过节假日
4. **限额检查**: 交易金额不能超过对手方授信额度
5. **汇率校验**: 外汇交易汇率需在合理范围内
6. **利息计算**: 定期存款按约定利率计算
7. **数据权限**: 用户只能查看/操作有权限的业务单元数据

## 五、验收标准

| 功能 | 验收条件 |
|------|----------|
| 新建交易 | 所有必填字段校验，提交成功 |
| 暂存 | 暂存后可在列表中找到，状态为草稿 |
| 交易复制 | 复制后生成新交易编号，相关字段复制 |
| 审批流转 | 多级审批正确流转，审批历史完整 |
| 状态变更 | 状态变更正确，消息通知到位 |
| 查询导出 | 多条件组合查询，导出Excel正常 |
| 交易变更 | 变更后生成变更记录，需审批 |
| 交易撤销 | 撤销后状态正确，记录撤销原因 |
| 权限控制 | 无权限数据不可见 |
