## 一、模块概述

**模块名称**: dealing - 资金交易管理  
**功能定位**: 支持手工交易录入、审批、查询，支持存款、贷款、外汇等交易类型  
**用户角色**: 交易员、交易主管、资金经理

---

## 二、交易类型定义

| 类型 | 子类型 | 说明 |
|------|--------|------|
| **存款** | 定期存款、活期存款、大额存单、通知存款 | 资金存放银行 |
| **贷款** | 短期贷款、中长期贷款 | 从银行借入资金 |
| **外汇** | 即期、远期、NDF、掉期、期权 | 外汇交易 |
| **同业** | 同业存放、存放同业、同业拆借 | 同业资金往来 |

## 四、业务规则

1. **编号规则**: 交易编号 = 业务单元编码 + 日期 + 序号，如 `BJ202604050001`
2. **金额校验**: 交易金额需大于0，精度匹配币种小数位数
3. **日期校验**: 起息日<=到期日，交割日需跳过节假日
4. **限额检查**: 交易金额不能超过对手方授信额度
5. **审批规则**: 可配置金额阈值，不同金额走不同审批流程
6. **利息计算**: 定期存款按约定利率计算，逾期按活期计息

## 五、验收标准

| 功能 | 验收条件 |
|------|----------|
| 新建交易 | 所有必填字段校验，提交成功 |
| 审批流 | 多级审批正确流转，审批历史完整 |
| 状态变更 | 状态变更正确，消息通知到位 |
| 查询 | 多条件组合查询，导出正常 |
| 变更/撤销 | 变更/撤销后状态正确，关联数据处理 |
| 复制 | 复制后生成新交易编号 |
