## 五、关键交互规范

### 5.1 金额输入交互

| 规范 | 内容 |
|------|------|
| 输入格式 | 每三位千分位分隔 (如: 1,234,567.89) |
| 货币符号 | 前置币种符号 (如: USD $1,234.56 / CNY ¥1,234.56) |
| 精度控制 | 按币种精度自动校验 (CNY:2位, JPY:0位) |
| 大小写转换 | 自动转换为大写金额显示 (可选) |
| 负数显示 | 括号形式 (¥100) 表示负数 |

### 5.2 多语言切换

| 规范 | 内容 |

| 切换方式 | 下拉选择，即时切换 |
| 生效范围 | 无需刷新，全界面生效 |
| 记忆方式 | 用户偏好持久化到用户配置 |

### 5.3 日期时间规范

## 十、后续模块设计说明

本设计方案为全局设计规范，各模块（M1-M5）需在此基础上进行细化设计。

**设计顺序：**
1. M1 基础数据模块 - 组织架构、权限、银行账户、交易录入
2. M2 资金运营模块 - 现金池、支付结算、存贷款
3. M3 金融工具模块 - 外汇、利率掉期、估值
4. M4 风险管理模块 - 敞口、VaR、套期保值
5. M5 分析报表模块 - 驾驶舱、报表

各模块设计时需遵循本规范的全局规则，同时补充模块特定的页面布局和交互细节。
