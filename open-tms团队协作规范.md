# Open-TMS AI团队协作 Prompt 模板

## 一、项目背景

**项目名称**: Open-TMS (Open Treasury Management System)
**项目目标**: 打造面向全球集团企业的企业资金管理系统，对标FIS Quantum、SAP资金管理
**核心价值**: 流动性管理、交易结算、敞口估值、风险控制

---

## 二、团队角色定义

| 角色 | 职责 | 汇报对象 |
|------|------|----------|
| **项目经理 (PM-Lead)** | 项目整体规划、进度管理、各角色协调、交付把关 | 用户 |
| **技术架构师 (TA)** | 技术选型、架构设计、关键技术方案、技术标准制定 | PM-Lead |
| **产品经理 (PM)** | 需求分析、功能设计、PRD撰写、验收 | PM-Lead |
| **开发 (Dev)** | 系统设计、代码实现、技术方案 | PM-Lead |
| **测试 (QA)** | 测试用例、测试执行、缺陷管理 | PM-Lead |
| **UX设计 (UX)** | 用户交互设计、界面原型、用户体验优化 | PM-Lead |

## 六、Prompt 使用指南

### 6.1 角色声明

每次分配任务时，需明确指定执行角色：

```
正确示例：
- "作为PM，请编写资金计划的PRD"
- "作为Dev，请实现交易录入API"
- "作为QA，请执行交易录入测试"
- "作为UX，请设计交易录入界面"

错误示例：
- "帮我写一下资金计划的PRD"
- "实现交易录入功能"
```

### 6.2 上下文传递

切换角色时，需简要说明背景：

```
"作为PM，上周已完成资金池需求评审，现在需要开发实现，请确认技术方案可行性。"
```

### 6.3 任务闭环

每个任务需有明确的结果：

```
PM: 输出PRD → Dev: 实现功能 → QA: 测试通过 → PM: 验收完成 → PM-Lead: 归档
```

### 7.1 GitHub Projects

**项目地址**: https://github.com/users/xiaoguo-jlu/projects/2

所有项目管理通过GitHub Projects进行，包括：Feature（需求）、US（用户故事）、Task（任务）、Bug（缺陷）。

### 7.2 工作项类型

| 类型 | 说明 | 状态流转 |
|------|------|----------|
| **Feature** | 新功能需求 | Todo → In Progress → Done |
| **US** | 用户故事 | Todo → In Progress → Done |
| **Task** | 技术任务 | Todo → In Progress → Done |
| **Bug** | 缺陷 | Todo → In Progress → Done |

### 7.3 字段说明

- **类型**: Feature / US / Task / Bug / 技术债务
- **优先级**: 高 / 中 / 低
- **Status**: Todo / In Progress / Done

### 7.4 工作流程

```
1. PM/PM-Lead 在GitHub Projects创建Issue
2. 设置类型（Feature/US/Task/Bug）和优先级
3. 分配给对应的角色（Dev/QA/UX）
4. 执行者将状态改为In Progress
5. 完成后再改为Done
6. 关联PR到Issue实现代码追溯
```

### 7.5 团队访问

团队成员使用自己的GitHub账号访问项目。进入项目页面 → Settings → Manage access → 添加成员账号。

#### 项目经理 (PM-Lead) 操作指南

```
## 表结构设计

### tms_bank_account_t
| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | UUID | 主键 | 自生成 |
| account_name | VARCHAR(100) | 账户名称 | 必填 |
| account_no | VARCHAR(50) | 账号 | 唯一 |
| bank_id | UUID | 所属银行 | 外键 |
| currency_code | CHAR(3) | 币种 | 外键 |

## 设计决策
1. 使用UUID作为主键，支持分布式场景
2. 账户号唯一性约束
3. 逻辑删除字段is_deleted

## 评审问题
- [ ] 是否需要支持多账号体系？
- [ ] 余额字段精度？
```
